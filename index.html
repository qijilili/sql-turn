转换规则说明

  • 表名（知识库优先）：命中 CSV 则以知识库为准。未命中且 CK/达芬奇→Hive：① 首个 zdm_ 改为 zdm_hive_（如 ads_zdm_trade_orderproduct→ads_zdm_hive_trade_orderproduct）；② 库名 = 表名中第一个 zdm 及之前字符（含 zdm），有 _hive_ 时优先用其左侧段定库名，避免误把 hive_zdm 当库名；再拼成 库名.表名。Hive→CK/达芬奇：表名去 _hive_，库名 bi_app。
  • 库名规范：正确形式均为 库名.表名（仅一段库名、一段表名，中间一个点）。CK/达芬奇库名为 bi_app；Hive 库名为表名中第一个 zdm 及其前字符。最终输出前会统一执行一次表名收口：先按知识库替换（若已加载），再按上述规则去 _hive_ / 补 zdm_hive_、合并误产生的多重库名、补全裸表名；与大模型或纯规则路径无关，保证结果表名一致规范。若仍疑似「库.库.表」会在结果末尾用注释提示。
  • CK / 达芬奇 — 表名以 _a 结尾：在 FROM / JOIN（含 LEFT JOIN 等）后的表引用后自动追加 final（如 bi_app.ads_zdm_foo_a → bi_app.ads_zdm_foo_a final）；已带 final 不重复。Hive 目标不追加（Hive 无此语法）。
  • Hive → 达芬奇： 先经大模型转为 CK，再应用库表名与达芬奇变量规则。时间：between、>= AND <=、以及 dt / order_create_date / statis_date / date = '日期' 等号写法，会转为 $begin_date$/$end_date$ 或 substr(...,1,7)；筛选项需在上方填写字段名
  • CK → 达芬奇： 规范 bi_app.表名 后同上时间规则与筛选项变量
  • 达芬奇 → Hive/CK： 变量替换为占位符后，转 Hive 调大模型并套表名规则；转 CK 规范为 bi_app
